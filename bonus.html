Бонусы: плагины, рой и готовые промпты

Собранные в одном месте: лучшие плагины под задачи, финальный workflow и библиотека bulletproof-промптов.

❙ Финальный workflow: от идеи до продакшена

Идея — Опиши задачу. Включи plan mode.
Интервью — Claude допрашивает → SPEC.md.
План — План по шагам, ты одобряешь.
Код — accept edits, тесты после каждого шага.
Ревью — /review + security-review.
Бэкап — Частые коммиты, /rewind как сеть.
Деплой — Хук/cron публикует и проверяет.
🔑
Это весь курс на одной странице. Каждый шаг опирается на свой модуль — возвращайся, когда нужно.

❙ Библиотека bulletproof-промптов

Готовые формы и intake — копируй и вставляй. Это самые надёжные шаблоны для работы с Claude Code.

Intake большой фичи (interview → SPEC)
Я хочу построить [описание]. Не пиши код. Сначала проведи
подробное интервью: техническая реализация, UI/UX, крайние
случаи, риски, компромиссы. По одному вопросу за раз, пока не
соберёшь полную картину. Затем запиши итог в SPEC.md.
Explore → Plan → Code → Commit
1) Изучи, как сейчас устроено [область], ничего не меняй.
2) Составь план изменений (plan mode).
3) Реализуй по шагам, после каждого запускай тесты.
4) Коммить логичными кусками с понятными сообщениями.
Bootstrap памяти проекта
Проанализируй проект и создай CLAUDE.md: команды запуска и
тестов, стиль кода, архитектурные решения, запретные зоны и
неочевидные гочи. Кратко, до 150 строк.
Security-ревью перед мержем
Проверь staged-изменения на уязвимости: инъекции (SQL/командные),
XSS, утечки секретов, дыры авторизации, path traversal, слабая
криптография. Верни находки по приоритету с конкретными фиксами.
Портрет целевой аудитории
По @README.md и @feedback.md собери 3 портрета ЦА: кто это,
их jobs-to-be-done, боли, возражения, триггеры покупки. Для
каждого — одна фраза ценностного предложения. Запиши в audience.md.
Цикл до зелёных тестов
/loop запусти npm test. Если падения — почини и запусти снова.
Остановись, когда все тесты пройдут.
