Модуль 06

CLAUDE.md — память проекта

Сделать так, чтобы Claude всегда помнил правила проекта: команды, стиль, архитектуру — без повторов.

06/20
30%
🎯 Результат модуля

У тебя есть CLAUDE.md, и Claude соблюдает правила в каждой сессии без напоминаний.

❙ Как работает память

CLAUDE.md — это markdown-файл, который Claude читает в начале каждой сессии. Туда кладут то, что он не угадает сам.

ФайлОбласть
./CLAUDE.mdПравила проекта (хранится в git, общий для команды).
~/.claude/CLAUDE.mdТвои личные предпочтения для всех проектов.
./CLAUDE.local.mdЛичное и секретное для этого проекта (в .gitignore).

❙ Создание за минуту

Выполни /init — Claude проанализирует проект и создаст черновик.
Добавляй факты на лету: начни строку с # — Claude сам запишет её в память.
Подключай большие документы через @import, чтобы не раздувать файл.
# Проект: Landing для онлайн-курса

## Команды
- `npm run dev` — запуск
- `npm test` — тесты (vitest)

## Стиль
- Только TypeScript, без any
- Компоненты — функциональные, Tailwind

## Важно
- Не трогать /legacy — старый код

См. @docs/architecture.md

❙ Правила хорошего CLAUDE.md

✗ Плохо

Длинно (500+ строк), пересказ очевидного, общие фразы вроде «пиши хороший код».

✓ Хорошо

< 200 строк, только неочевидное: команды, гочи, запреты, конкретные конвенции.

💡
Чем короче и конкретнее файл, тем лучше Claude его соблюдает. Длинные простыни он начинает игнорировать.
