Модуль 11

Subagents — параллельные исполнители

Делегировать исследование и подзадачи отдельным агентам со своим контекстом и инструментами.

11/20
55%
🎯 Результат модуля

Ты создаёшь субагентов и понимаешь, зачем им отдельное окно контекста.

Простыми словами

Субагент — помощник с отдельной головой: поручаешь ему рутину (поиск, проверку), а он приносит готовый результат, не засоряя твой разговор.

❙ Зачем нужны субагенты

Субагент — это «помощник» со своим окном контекста, набором инструментов и даже своей моделью. Он делает грязную работу (поиск, исследование, ревью), а в основной диалог возвращает только вывод. Контекст главного агента остаётся чистым.

Чистота

Тяжёлое исследование не засоряет основной чат.

context

Ограничения

Даёшь агенту только нужные инструменты.

tools

Экономия

Дешёвая модель (haiku) на простой поиск.

model

Параллель

Несколько агентов работают одновременно.

parallel

❙ Создание субагента

Запусти /agents для интерактивного создания — или положи markdown-файл в .claude/agents/.

---
name: security-reviewer
description: Аудит кода на уязвимости перед мержем
tools: [Read, Grep, Bash]
model: opus
---
Ты — senior security-инженер. Ищи: SQL/XSS-инъекции,
утечки секретов, дыры в авторизации, path traversal,
слабую криптографию. Возвращай находки с приоритетом.
🔑
Правило: один субагент — одна узкая роль. «Швейцарский нож» работает хуже, чем три заточенных агента.
