Модуль 16

Loops — повторение до результата

Запускать промпт по кругу: следить за деплоем, опрашивать статус, доводить задачу до готовности.

16/20
80%
Результат модуля

Ты используешь /loop для мониторинга и итеративных задач без ручного повтора.

Простыми словами

Loop — повтор по кругу: «проверяй каждые 5 минут» или «чини тесты, пока не станут зелёными». Вручную нажимать не нужно.

❙ Что такое loop

Loop — это повторяющийся запуск промпта на интервале. Удобно для мониторинга («проверяй деплой каждые 5 минут») и для итераций до выполнения условия.

# фиксированный интервал
/loop 5m проверь, закончился ли деплой, и сообщи статус

# динамический — Claude сам выберет частоту
/loop следи за статусом PR и новыми комментариями ревью

❙ Цикл до условия

Мощный паттерн — крутить цикл, пока цель не достигнута: «запускай тесты и чини падения, пока всё не станет зелёным».

/loop запусти npm test. Если есть падения — почини и запусти снова. Останавливайся, когда все тесты пройдут.
Цикл крутит реальные действия и тратит токены. Ставь чёткое условие остановки и поглядывай на /cost. Прервать ожидание — Esc.

Сделай сам

Запусти loop «запускай тесты и чини, пока не зелёные» на реальном куске кода.
