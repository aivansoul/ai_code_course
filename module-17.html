Модуль 17

Hivemind — мульти-агентная оркестрация

Понять «рой» агентов: как запускать много Claude параллельно над частями задачи и собирать результат.

17/20
85%
🎯 Результат модуля

Ты понимаешь концепцию hivemind и официальные способы параллельной работы.

❙ Что такое hivemind

Hivemind («рой», swarm) — это паттерн, где над одной большой задачей работает много агентов параллельно: один распределяет («queen»/оркестратор), другие выполняют куски («workers»), результаты сводятся вместе. Это даёт охват и скорость, которые не вместить в один контекст.

⚠️
Важно по-честному: «queen + workers» как готовая кнопка — это сообществ­ные надстройки, не официальная фича. Официально hivemind собирается из реальных примитивов ниже.

❙ Официальные кирпичики роя

Субагенты

Несколько исполнителей параллельно, каждый — свой контекст (модуль 11).

parallel

Worktrees

Отдельная git-ветка-копия под каждого агента — правки не конфликтуют.

git

Background tasks

Фоновые сессии, за которыми следишь, пока делаешь своё.

async

Оркестратор

Главный агент дробит задачу, раздаёт и сводит результаты.

queen

❙ Паттерн на практике

Разбей задачу на независимые куски (например, 5 модулей рефакторинга).
Под каждый — свой git worktree, чтобы правки не пересекались.
Запусти агента в каждом; оркестратор сводит и проверяет результат.
git worktree add ../task-auth   -b task-auth
git worktree add ../task-billing -b task-billing
# в каждой папке — свой claude, потом смержить ветки
🔑
Сообществ­ные фреймворки (напр. claude-flow от ruvnet) автоматизируют рой: оркестратор-«queen», воркеры и общая память. Полезно знать, но начинай с официальных субагентов и worktrees.
