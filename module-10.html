Модуль 10

Skills — навыки, которые Claude подключает сам

Научить Claude «как думать» в типовых задачах: skill подгружается автоматически, когда нужен.

10/20
50%
🎯 Результат модуля

Ты умеешь создавать SKILL.md и понимаешь, чем skill отличается от plugin и subagent.

Простыми словами

Skill (навык) — сохранённая инструкция «как делать такую задачу». Claude сам достаёт нужный навык, когда задача подходит — как шпаргалка под рукой.

❙ Что такое Skill

Skill — это инструкция-навык в файле SKILL.md. Claude сам решает подключить её, когда задача подходит под описание. В отличие от CLAUDE.md (грузится всегда), skill подгружается по требованию — контекст остаётся чистым.

ПонятиеСуть
Skill«Как думать» в задаче. Инструкция, подключается сама.
SubagentОтдельный исполнитель со своим контекстом (модуль 11).
PluginПакет: команды + агенты + MCP + хуки (модуль 12).

❙ Создание навыка

---
name: code-review
description: Ревью изменений на баги и читаемость. Использовать перед коммитом фичи.
---

# Как делать ревью
1. Сначала пробеги git diff.
2. Ищи: краевые случаи, утечки, дубли, плохие имена.
3. Возвращай список находок с приоритетом и фиксами.
Ключевое — поле description: по нему Claude решает, когда подключить навык.
Создай папку .claude/skills/имя/ и файл SKILL.md.
Опиши в description, когда навык применять.
Проверь: дай задачу под навык — Claude подключит его сам.
💡
Навыки бывают личные (~/.claude/skills), проектные (.claude/skills) и из плагинов. Личные едут с тобой во все проекты.
